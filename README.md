## ✨ Key Features

1. **🚦 Dual-Model Traffic Detection**:
   - **Primary Model** (`best.pt`): Detects motorcycles, helmets, bare heads (`no-helmet`), and number plates.
   - **TVD Model** (`best_tvd.pt`): Detects advanced violations including `triple riding`, `using mobile while driving`, and `wheeling`.
   - Results from both models are merged automatically using intersection-over-union (IoU) deduplication.

2. **🔍 Intelligent License Plate Reader (OCR)**:
   - High-accuracy license plate extraction.
   - Multi-engine flexibility using **EasyOCR** or **PaddleOCR** with automatic validation against standard Indian state RTO codes.

3. **🧠 RAG-Grounded Bilingual E-Challan**:
   - Pulls exact sections, fines, and disqualification rules from a local ChromaDB index storing the **Motor Vehicles Act 1988 (Amended 2019)**.
   - Grounded LLM generation (using Groq Llama-3.3) outputs formal challans in both **English** and **Hindi/Devanagari** in under 2 seconds.
   - *Offline Fallback*: If API keys are missing or offline, the app falls back on a local static template to guarantee uptime.

4. **📊 Real-time Enforcement Analytics**:
   - Visual charts showing violation types, OCR engine distribution, plate formats, and cumulative fines collected.

5. **📁 SQLite Persistence & History Log**:
   - All violations, crop hashes, confidence scores, and raw timestamps are saved in `violations.db`.
   - Includes a searchable history grid where you can instantly review annotated violation evidence.

6. **💬 Legal AI Chatbot**:
   - Ask questions about the Motor Vehicles Act (e.g., *"What is the fine for triple riding?"*).
   - The chatbot uses RAG to fetch matching laws, citing sections and compoundability.

> **Note**: The TVD model's "Using Mobile" class has lower precision due to limited training samples. The dual-model merge strategy compensates — the primary model handles core detections (helmet, plate, bike) with high accuracy, while the TVD model adds supplementary violation coverage for triple riding, mobile use, and stunts.

### 🔗 Dual-Model Merge Strategy

The system runs **both models in parallel** on every uploaded image and merges results using IoU-based deduplication:
- Primary model provides the backbone — bikes, helmets, bare heads, and plates with **94% mAP**.
- TVD model adds specialized violation categories — particularly **wheeling (99.5% mAP)** and **no-helmet (89.4% mAP)**.
- Overlapping detections (IoU > 0.5) are deduplicated, keeping the higher-confidence prediction.
- Triple riding is additionally inferred by counting rider heads per bike (>2 heads = triple riding), providing a fallback even if the TVD model misses it.

## 🛠️ Step-by-Step Setup Guide (For Non-Technical Users)

*Follow these simple instructions to set up and run the application on your computer.*

### 1️⃣ Install Prerequisites (One-Time Setup)
Make sure you have the following installed on your computer:
1. **Python (3.10 or higher)**: [Download here](https://www.python.org/downloads/)
   * *⚠️ Important: During installation, check the box that says **"Add Python to PATH"** before clicking install.*

2. **Node.js (LTS version)**: [Download here](https://nodejs.org/en/download/)
   * *Just download the recommended "LTS" installer and click "Next" through the setup.*

---

### 2️⃣ Install the App
1. Open the search bar on your computer (press the Windows key) and type **Command Prompt** (or **Terminal** on Mac/Linux), then open it.
2. Navigate to this project folder. For example, if it is in `D:\grid-traffic`, type:
   ```cmd
   D:
   cd D:\grid-traffic
   ```
3. Copy and paste the following commands one by one to install all requirements:
   ```cmd
   :: 1. Create a virtual python environment
   python -m venv venv

   :: 2. Activate the environment
   :: On Windows:
   call venv\Scripts\activate
   :: On Mac/Linux:
   source venv/bin/activate

   :: 3. Install backend dependencies
   python -m pip install --upgrade pip
   pip install -r requirements.txt

   :: 4. Install frontend dependencies
   cd web
   npm install
   cd ..
   ```

---

### 3️⃣ Check API Keys
The application is pre-configured with active demonstration keys in the `.env` file. If you ever need to use your own keys:
1. Open the `.env` file in the project folder with Notepad.
2. Enter your Groq API key:
   ```env
   GROQ_API_KEY=your_actual_key_here
   ```
3. Save and close the file.

---

### 4️⃣ Start the Servers
To run the app, you need to start the backend and the frontend. 

1. **Start the Backend (API Server)**:
   - Open a **Command Prompt** and navigate to your folder:
     ```cmd
     D:
     cd D:\grid-traffic
     ```
   - Run:
     ```cmd
     :: Activate the virtual environment
     call venv\Scripts\activate
     
     :: Start backend
     uvicorn server:app --host 0.0.0.0 --port 8000
     ```
   - *Keep this terminal window open.*

2. **Start the Frontend (User Interface)**:
   - Open a **second, separate Command Prompt** and navigate to the `web` folder:
     ```cmd
     D:
     cd D:\grid-traffic\web
     ```
   - Run:
     ```cmd
     npm run dev
     ```
   - *Keep this terminal window open.*

3. **Open the Webpage**:
   - Open your web browser and go to: **http://localhost:3000**
   - 🎉 **You're ready to use the app!**

## ⚙️ How it Works under the Hood

1. **Dual YOLO Model Inference**:
   We merge standard YOLO detections with a dedicated *Traffic Violation Detection (TVD)* model. If a rider is caught without a helmet, triple riding, wheeling, or texting while driving, the violation logic instantly flags the vehicle.
2. **Text OCR Parsing**:
   The number plate region is cropped and passed to our OCR pipeline. It standardizes whitespace, filters non-alphanumeric characters, and recognizes standard Indian formats (e.g., `KA01AB1234`).
3. **Retrieval-Augmented Generation (RAG)**:
   The list of violations (e.g., `no_helmet`, `triple_riding`) is passed to ChromaDB. ChromaDB queries semantic vector indexes to find the exact penalty sections from the Motor Vehicles Act 1988.
4. **Bilingual E-Challan Output**:
   Llama-3.3 receives the exact law text. It formats the official citation into a dual English-Hindi receipt, ensuring absolute citation accuracy with zero model hallucination.
